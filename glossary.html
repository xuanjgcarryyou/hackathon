③ 容器循環流程
英文 / 代碼中文說明類型
Dispatch出貨／配送將循環容器從廠商送至企業的動作，產生一筆 ContainerBatch動作
Collect回收員工掃 QR Code 後登記容器歸還，更新 collected_count動作
QR Code二維碼每個 ContainerBatch 的唯一識別標籤，用於掃描回收欄位
status: dispatched已出貨容器已送出、尚未回收狀態值
status: collected已回收容器已全數或部分歸還狀態值
quantity出貨數量一批次配送的容器總數欄位
collected_count回收數量實際歸還的容器數，≤ quantity欄位
returnRate回收率collected_count ÷ quantity，0–1 的小數指標
anomaly異常回收率低於 anomalyThreshold（預設 0.9）時觸發的警示旗標旗標
ANOMALY_THRESHOLD異常閾值預設 0.9（90%），低於此值則廣播 anomaly_alert 事件常數
PACKAGING_KG_PER_UNIT每份包材重量每個容器預設 0.025 kg，用於計算減少包材總重常數
④ 碳排計算與因子
英文 / 代碼中文說明類型
CO₂e二氧化碳當量統一換算不同溫室氣體影響的排放單位（kg CO₂e）單位
co2eSaved碳排避免量使用循環容器相較一次性包材所減少的 kg CO₂e（Avoided Emissions）指標
carbonFactor碳因子每個容器每次使用的 kg CO₂e，預設 0.15欄位
carbonFactorSource碳因子來源說明係數出處（如 ecoinvent、工研院）的文字欄位欄位
reducedPackagingKg減少包材重量circlularMeals × PACKAGING_KG_PER_UNIT，單位 kg指標
Lifecycle Stage生命週期階段一次性：原料→製造→運輸→廢棄；循環：生產→正向配送→清洗→逆物流→損耗概念
raw_material原料取得一次性包材生命週期第一階段階段
manufacturing製造加工一次性包材生命週期第二階段階段
disposal廢棄處理一次性包材生命週期最終階段階段
washing清洗消毒循環容器每次使用後的清潔步驟，含碳排階段
reverse_logistics逆物流容器從企業回到廠商清洗設施的運輸階段
damage_loss損耗攤提容器報廢成本按使用次數均攤的碳排階段
expectedLifespanCycles預期使用壽命循環容器在報廢前預計可使用的總次數欄位
confidenceLevel信心水準碳排數據品質：low / medium / high列舉
⑤ ESG 報告框架
英文 / 代碼中文說明類型
ESG環境、社會、治理Environmental, Social, Governance 永續發展框架標準
GHG Protocol溫室氣體盤查協定全球最通用的企業碳排計算標準，本平台依此分 Scope 1/2/3標準
ISO 14064-1組織碳盤查標準GHG Protocol 對應的 ISO 標準，報告封面標示使用標準
Scope 1範疇一直接排放（燃料燃燒、逸散），本平台標示「不適用」範疇
Scope 2範疇二電力間接排放，本平台標示「未追蹤」範疇
Scope 3 Cat.1範疇三類別一採購商品與服務：本平台的主要揭露項目（包材 Avoided Emissions）範疇
Scope 3 Cat.4範疇三類別四上游運輸與配送，標示「未追蹤」範疇
Scope 3 Cat.5範疇三類別五營運廢棄物，標示「未追蹤」範疇
Avoided Emissions避免排放量循環容器取代一次性包材所「避免」產生的排放，為補充揭露指標概念
dataHash資料稽核碼對原始 BatchRecord 計算的 SHA-256，用於第三方稽核驗真欄位
reportTextZh / EnAI 報告文本（中／英）由 Claude AI 生成的完整敘事性 ESG 報告段落欄位
isFallback本地備援旗標Claude API 無回應時改用本地模板，此旗標為 true 並顯示警示橫幅旗標
totalMeals總訂餐份數報告期間內所有已收回批次的容器總數指標
circularMeals循環容器份數實際回收的容器總數（summed collected_count）指標
